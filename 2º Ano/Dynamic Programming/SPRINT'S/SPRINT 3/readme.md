# Dynamic Programming - Sprint 3

## Objetivo
Implementar soluções utilizando recursão e memoização para resolver problemas de duplicidade de leads e otimização de agenda em um CRM hospitalar.

## Funcionalidades

### Verificação de duplicidade
Função recursiva que percorre a lista de leads e verifica se um novo cadastro já existe com base em CPF ou e-mail.

### Memoização
Utilização de cache para evitar comparações repetidas, melhorando a eficiência da verificação.

### Otimização de agenda
Função recursiva que verifica combinações de horários disponíveis para encaixar consultas de forma eficiente.

## Resultado
O sistema consegue identificar duplicidades corretamente e otimizar o processo utilizando memoização, reduzindo o número de operações necessárias.
